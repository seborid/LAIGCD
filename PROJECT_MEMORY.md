# LAIGCD 项目记忆

更新日期：2026-06-10

## 1. 这是什么项目

项目正式题目是：

- **基于空域语义分析和频域分析的轻量化、可解释性 AIGC 人脸检测系统**

项目应按两个部分理解：

- **第一部分：真伪判断**
  - 目标：判断输入人脸图像是真实还是 AIGC / deepfake 伪造
  - 当前主实现：`CLIP ViT-B/32` 空域语义特征 + `SRM` 频域分支 + 原型学习分类头
- **第二部分：多模态可解释性**
  - 目标：在真假判断结果之外，结合原图、热力图、频域响应等信息生成自然语言解释
  - 当前状态：已有设计文档和思路，但尚未形成完整可运行主链路

LAIGCD 的实现已经明显收敛到一个更具体的方向：

- **当前真实定位**：轻量级 **AIGC / Deepfake 人脸检测系统**
- **当前主要数据源**：
  - `140k-real-and-fake-faces`
  - `130k-real-vs-fake-face`
- **当前核心方法**：
  - 冻结的 `CLIP ViT-B/32` 空域特征
  - `SRM` 频域分支
  - `PrototypeModule` 原型学习分类头

结论：后续讨论项目时，应把它视为“**AIGC 人脸检测 + 多模态可解释性**”的两阶段系统；其中第一阶段代码已基本落地，第二阶段仍主要停留在设计与扩展阶段。

## 4. 当前实验快照

以下结论以 `checkpoints/` 下现有配置和日志为准。

### quick_run

- 配置：`max_samples=1000`，`epochs=5`
- 最佳 AP：`0.5057`
- Accuracy：长期 `0.5000`
- 含义：代码能跑通，但模型效果几乎未形成

### standard_run

- 配置：`max_samples=10000`，`epochs=15`
- 最佳 AP：`0.9463`
- Accuracy：长期 `0.5000`
- 含义：排序能力已经很强，但默认阈值下分类决策明显异常

### full_run

- 配置：全量数据，`epochs=30`，`batch_size=64`
- 最佳 AP：`0.9997`
- Accuracy：约 `0.5003`
- 含义：模型几乎肯定已经学到很强的区分排序能力，但当前 **0.5 阈值不可用**，项目还不能把这个结果直接当成“分类器已经完成”

### 数据与产物体量

- `data/140k-real-and-fake-faces`：约 `4.1G`
- `data/130k-real-vs-fake-face`：约 `13G`
- `checkpoints/`：约 `21G`

## 6. 已识别的明显风险/问题

### 代码/脚本问题

- `scripts/eval.py` 与当前 dataloader 接口不一致
  - 代码里按 `(images, labels)` 解包
  - 但 `AIGCDataset` 返回的是字典
- `scripts/eval.py` 使用 `split=val` 的方式，也和当前 `get_val_dataloader()` 接口不匹配
- `scripts/train.sh` 传参方式可疑
  - `--use_freq "$USE_FREQ"` 不符合 `argparse` 里布尔 flag 的定义
- `scripts/train_optimized.sh` 在 `MAX_SAMPLES` 非空时命令拼接有问题
  - 会把 `--max_samples ...` 拼到命令最前面
- `test/` 目录存在，但目前没有有效自动化测试

### 文档问题

- `README.md` 仍把项目写成更通用的 AIGC 检测器
- `TODO.md` 大量事项仍显示未完成，但代码其实已实现不少
- 计划文档、设计文档、真实实现之间已经有明显漂移
